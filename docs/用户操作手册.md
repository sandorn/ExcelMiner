# ExcelMiner 用户操作手册

> 版本：v0.6.0 | 更新日期：2026-05-29

---

## 一、产品概述

ExcelMiner（AardMiner）是一款 Windows 桌面应用，用于多子公司 Excel 经营数据的自动汇总与 DeepSeek AI 经营分析。它将原本需要手动操作 VBA 宏的繁琐流程自动化，提供从数据汇总到 AI 分析再到报表导出的一站式体验。

### 核心功能

| 功能         | 说明                                                               |
| ------------ | ------------------------------------------------------------------ |
| 数据汇总     | 自动识别活动量和经营报表文件夹中的 Excel 文件，按业态汇总 YTD 指标 |
| AI 经营分析  | 调用 DeepSeek 大模型，按板块和公司生成专业经营分析报告             |
| 报表导出     | 将汇总数据和分析结果写入标准格式的 xlsx 模板文件                   |
| PPT 文案生成 | 一键复制分析文案到剪贴板，可直接粘贴到 PPT                         |

### 覆盖范围

系统管理 **9 家子公司**，分为 **3 个业态**：

| 业态 | 子公司                                           |
| ---- | ------------------------------------------------ |
| 保险 | 盛唐融信、君康经纪                               |
| 酒店 | 伯豪瑞廷、重庆瑞尔                               |
| 商写 | 北京中言、大连凯丹、福建钱隆、春夏秋冬、重庆宜新 |

## 二、安装与启动

> 系统要求详见 [README.md](../README.md#系统要求)。

### 2.1 获取程序

从发布目录获取 `release-portable/` 文件夹，双击 `ExcelMiner.exe` 即可启动。

> 便携版无需安装，可直接拷贝到任意目录运行。

### 2.2 配置 API Key

程序需要 DeepSeek API Key 才能使用 AI 分析功能。有两种配置方式：

**方式一：~/.dskey 文件（推荐）**

在用户目录下创建 `C:\Users\<用户名>\.dskey` 文件，内容如下：

```ini
[EXCEL]
key = sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

程序启动时会自动读取，无需每次手动输入。

**方式二：界面输入**

在 AI 分析步骤的 API Key 输入框中直接粘贴。

### 2.3 界面布局

程序采用单页面布局，上方为配置区，下方为日志输出区，中间为三个执行按钮：

```
┌──────────────────────────────────────────────┐
│  AardMiner  经营数据汇总分析工具              │
├──────────────────────────────────────────────┤
│  配置区：年月 | 数据目录 | 结果文件 | API Key  │
│  模型：模型选择                               │
├──────────────────────────────────────────────┤
│  [一键汇总]  [板块业态分析]  [公司经营分析]    │
├──────────────────────────────────────────────┤
│  日志输出区（实时滚动）                       │
└──────────────────────────────────────────────┘
```

## 三、操作流程

整个工作流程分为三个步骤，按顺序执行：

```
Step 1: 配置项目 → Step 2: 数据汇总 → Step 3: AI 分析
```

### Step 1：配置项目

#### 1.1 参数说明

| 参数       | 说明                                                       | 必填 |
| ---------- | ---------------------------------------------------------- | ---- |
| 报告年月   | 分析报告的目标月份，如"2026年4月"                          | 是   |
| 数据源目录 | 包含 `活动量/` 和 `经营报表/` 子文件夹的目录               | 是   |
| 结果文件   | 输出 xlsx 文件的路径（通常选择数据源目录下的经营数据文件） | 是   |
| API Key    | DeepSeek API 密钥（可从 ~/.dskey 自动读取）                | 否\* |
| 模型       | AI 模型选择，默认 `deepseek-chat`                          | 否   |

> \* API Key 在 Step 3 执行 AI 分析前配置即可。

#### 1.2 数据目录结构

数据源目录必须遵循以下规范：

```
数据源目录/
├── [YYYY年M月] 经营数据.xlsx   ← 汇总输出模板文件
├── 活动量/                        ← 营销活动源文件
│   ├── 盛唐融信.xlsx
│   ├── 君康经纪.xlsx
│   ├── 北京中言.xlsx
│   ├── 大连凯丹.xlsx
│   ├── 福建钱隆.xlsx
│   ├── 春夏秋冬.xlsx
│   ├── 重庆宜新.xlsx
│   ├── 伯豪瑞廷.xlsx
│   └── 重庆瑞尔.xlsx
└── 经营报表/                      ← 经营指标源文件
    ├── 盛唐融信.xlsx
    ├── 君康经纪.xlsx
    ├── 北京中言.xlsx
    ├── 大连凯丹.xlsx
    ├── 福建钱隆.xlsx
    ├── 春夏秋冬.xlsx
    ├── 重庆宜新.xlsx
    ├── 伯豪瑞廷.xlsx
    ├── 重庆瑞尔.xlsx
    └── 上海联杰.xlsx
```

#### 1.3 操作步骤

1. 点击"数据源目录"右侧的文件夹图标，选择数据源目录
2. 点击"结果文件"右侧的文件夹图标，选择输出 xlsx 模板文件
3. 选择报告年月
4. 确认 API Key 已配置（显示"已配置"标签）
5. 点击"保存配置"按钮

> 如果之前创建过项目，可以点击"打开项目"按钮，选择 `.project.toml` 文件恢复配置。

---

### Step 2：数据汇总

#### 2.1 汇总引擎

系统内置 4 个汇总引擎，按固定顺序执行：

| 引擎     | 说明                                   | 数据源位置  |
| -------- | -------------------------------------- | ----------- |
| 经营报表 | 复制各公司经营指标的原始数据到输出文件 | `经营报表/` |
| 保险汇总 | 汇总盛唐融信、君康经纪的保险业务指标   | `活动量/`   |
| 商写汇总 | 汇总 5 家商写公司的招商运营指标        | `活动量/`   |
| 酒店汇总 | 汇总伯豪瑞廷、重庆瑞尔的酒店经营指标   | `活动量/`   |

#### 2.2 汇总指标示例

**保险业态** 汇总指标包括：

- 期初/月末人力、入职/离职人数
- 承保新单规模保费 YTD、承保期交规模保费 YTD
- 续期 13 月/25 月应收实收
- 开单人数、承保件数
- 派生指标：活动率、件均保费、人均保费

**商写业态** 汇总指标包括：

- 新签面积/合同数、续签面积/合同数
- 在租面积/合同数
- 招商渠道分布（代理行、自有渠道等）

**酒店业态** 汇总指标包括：

- 营销活动场次/人次
- OTA 评分、入住率、平均房价（ADR）
- RevPAR、GOP 率

#### 2.3 操作步骤

1. 确认配置已保存
2. 点击 **"一键汇总"** 按钮
3. 观察日志输出区的进度信息
4. 等待提示"汇总完成"，显示耗时

> 汇总过程通常可在 1 分钟内完成，取决于文件数量。

---

### Step 3：AI 分析

AI 分析分为两个阶段，**必须先执行板块分析，再执行公司分析**：

#### 3.1 阶段一：板块业态分析

对保险、酒店、商写三个业态分别生成板块级经营分析。

| 板块 | 数据来源               | 系统提示词文件        |
| ---- | ---------------------- | --------------------- |
| 保险 | 保险汇总引擎的汇总数据 | `prompts/保险分析.md` |
| 酒店 | 酒店汇总引擎的汇总数据 | `prompts/酒店分析.md` |
| 商写 | 商写汇总引擎的汇总数据 | `prompts/商写分析.md` |

> 板块分析**不进行质量评分**，仅验证内容长度 ≥ 50 字。

#### 3.2 阶段二：子公司经营指标分析

逐一对 9 家子公司进行经营指标分析，使用财务分析师提示词。

**质量评分体系**（满分 8 分）：

| 维度       | 满分 | 说明                                |
| ---------- | ---- | ----------------------------------- |
| 营业收入   | 2    | 是否准确描述收入数据及同比/环比变化 |
| EBITDA/GOP | 2    | 是否准确描述 EBITDA/GOP/扣非净利润  |
| 现金流     | 2    | 是否准确描述现金流情况              |
| 费用管控   | 2    | 是否准确描述费用变化及管控措施      |

- 质量得分 ≥ 8 分 → 通过
- 质量得分 < 8 分 → 自动重试（最多 2 次），每次附带质量改进提示
- 摘要部分不计入评分

#### 3.3 操作步骤

1. **先执行板块分析**：点击 **"板块业态分析"** 按钮
2. 等待三个板块分析完成（日志会显示各板块耗时）
3. **再执行公司分析**：点击 **"公司经营分析"** 按钮
4. 等待 9 家公司逐一分析完成（并发执行，通常 1-2 分钟）

> 公司分析使用 18 路并发 + 全并发执行，效率较高。如果某家公司质量不达标会自动重试。

## 四、查看结果

### 4.1 日志输出

所有操作过程都会实时输出到底部日志区，包括：

- 每个引擎的汇总进度
- 每家公司/板块的 AI 分析进度
- 质量评分和重试信息
- 总耗时统计

### 4.2 导出报表

分析完成后，汇总数据和 AI 分析内容会写入输出 xlsx 模板文件的对应位置。直接用 Excel 打开结果文件即可查看。

## 五、项目文件管理

### 5.1 项目配置文件

程序会在数据源目录同级生成 `.project.toml` 文件，保存项目配置：

- 项目名称、年月
- 数据目录、输出文件路径
- AI 配置（模型、API URL 等）
- 公司列表

### 5.2 打开已有项目

下次打开程序时，点击"打开项目"按钮，选择之前生成的 `.project.toml` 文件即可恢复所有配置。

### 5.3 日志文件

详细日志保存在 `%APPDATA%/ExcelMiner/logs/app.log`，可用于排查问题。

## 六、常见问题

### Q1：提示"API Key 未配置"

检查 `C:\Users\<用户名>\.dskey` 文件是否存在且格式正确。或者直接在界面 API Key 输入框中粘贴 Key，然后点击保存配置。

### Q2：汇总时提示"找不到文件"

检查数据源目录结构是否符合规范（必须有 `活动量/` 和 `经营报表/` 子文件夹），确保公司 Excel 文件名与模板名称一致。

### Q3：AI 分析结果质量不达标

系统会自动重试最多 2 次。如果 2 次重试后仍不达标，该公司的分析结果仍会保留，可手动调整。质量评分仅用于内部校验，不影响最终输出。

### Q4：如何修改 AI 提示词

提示词文件位于程序目录的 `resources/prompts/` 下：

- `保险分析.md` — 保险板块分析提示词
- `酒店分析.md` — 酒店板块分析提示词
- `商写分析.md` — 商写板块分析提示词
- `财务分析师.md` — 公司经营指标分析提示词

直接编辑对应 Markdown 文件即可自定义提示词内容，下次分析时自动生效。

### Q5：程序窗口太小

窗口支持自由缩放，最小尺寸 1024×700。也可以拖动窗口边缘调整大小。

### Q6：如何获取 DeepSeek API Key

访问 [platform.deepseek.com](https://platform.deepseek.com) 注册账号，在 API Keys 页面创建密钥。

## 七、附录：配置参数速查

| 参数              | 默认值                                         | 说明                      |
| ----------------- | ---------------------------------------------- | ------------------------- |
| API URL           | `https://api.deepseek.com/v1/chat/completions` | DeepSeek API 地址         |
| Model             | `deepseek-chat`                                | AI 模型名称               |
| Temperature       | 0.3                                            | 生成温度（0-2）           |
| Max Tokens        | 1500                                           | 单次最大输出 token 数     |
| Batch Size        | 3                                              | 批量分析大小              |
| Max Retries       | 2                                              | 质量不达标最大重试次数    |
| Quality Threshold | 8                                              | 质量评分通过阈值（满分8） |
| HTTP Timeout      | 60s                                            | API 请求超时时间          |
