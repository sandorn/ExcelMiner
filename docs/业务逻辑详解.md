# 业务逻辑详解

> 本文档完整描述从 Excel 源文件到最终分析文案的全链路业务处理逻辑，不涉及技术实现细节。

---

## 一、项目概述

这是一个经营数据汇总分析工具。它从分散的 Excel 数据源中汇总各子公司的经营指标，通过 AI 大模型（DeepSeek）生成板块级和公司级的经营分析文案，最终输出到格式化的 Excel 工作簿中。

**核心流程**分为三个阶段：

1. **数据汇总** — 从"活动量"和"经营报表"两个文件夹读取各公司 Excel 源文件，按业态汇总计算后填入输出工作簿的对应 Sheet
2. **板块 AI 分析** — 对商写、保险、酒店三个业态分别生成分析文案
3. **公司 AI 分析** — 对全部 9 家子公司逐家生成经营指标分析文案

## 三、关键业务参数

以下参数在输出工作簿的"填写页"中配置，部分也支持在项目配置文件中预设：

| 参数       | 位置            | 说明                                              |
| ---------- | --------------- | ------------------------------------------------- |
| 报告月份   | `填写页!A2`     | 如 `2026年4月` → 提取数字 4，支持日期格式         |
| 报告年份   | `填写页!A4`     | 用于 AI 分析输出的年份标注                        |
| 序时进度   | `填写页!A8`     | 如 33.33%（4/12），用于经营指标分析中的达成率对比 |
| 公司列表   | `填写页!C2:C10` | 经营报表汇总据此遍历公司                          |
| 分析提示词 | `填写页!C20`    | 经营指标分析的系统提示词                          |

**月份自动检测优先级**：

1. 读取输出文件的"填写页!A2"单元格
2. 从数据源目录名解析（如 `2026年4月` → 4）
3. 回退到系统当前月份

## 四、数据汇总

数据汇总阶段按业态顺序执行：经营报表复制 → 保险汇总 → 商写汇总 → 酒店汇总。

### 4.1 数字提取规则

系统从 Excel 单元格中提取数值时遵循以下规则：

| 优先级 | 输入情况    | 处理逻辑                               | 结果   |
| ------ | ----------- | -------------------------------------- | ------ |
| 1      | 空单元格    | 返回 0                                 | 0      |
| 2      | 纯数值      | 直接取值                               | 12.5   |
| 3      | 文本含数字  | 逐字符扫描：收集连续数字和第一个小数点 | 见示例 |
| 4      | 纯中文/符号 | 无数字 → 返回 0                        | 0      |

示例：

- `"直播1736场"` → 提取 `1736`
- `"3.5万"` → 提取 `3.5`
- `"1+1000"` → 分割求和 = `1001`（酒店业态专用）
- `"85%"` → 提取 `85`（不处理百分号）
- `"¥1,234.56"` → 提取 `1234.56`

> 酒店业态的营销活动单元格支持加法表达式求和（如 `"1+1000"` → `1001`），其余业态使用基础版提取。

### 4.2 保险数据汇总

**数据源**：`活动量/{公司}.xlsx` → Sheet `保险类`

**汇总公司**：盛唐融信、君康经纪（2 家）

**目标位置**：输出文件 `保险类` Sheet

#### 汇总指标

| 指标                 | 计算方式                                     |
| -------------------- | -------------------------------------------- |
| 期初人力             | 直接取 1 月达成值                            |
| 当月入职（YTD）      | 1 月至报告月达成列逐月累加                   |
| 当月离职（YTD）      | 同上                                         |
| 当月净增             | = YTD入职 − YTD离职                          |
| 月末人力             | = 期初人力 + YTD入职 − YTD离职               |
| 平均人力             | = Σ(各月月末人力) ÷ 报告月数（逐月递推计算） |
| 开单人数（YTD）      | 1 月至报告月逐月累加                         |
| 承保新单规模保费 YTD | 1 月至报告月逐月累加                         |
| 承保期交规模保费 YTD | 1 月至报告月逐月累加                         |
| 续期 13 月应收       | 报告月当月值，不累加                         |
| 续期 13 月实收       | 报告月当月值，不累加                         |
| 续期 25 月应收       | 报告月当月值，不累加                         |
| 续期 25 月实收       | 报告月当月值，不累加                         |
| 承保件数（YTD）      | 1 月至报告月逐月累加                         |

#### 派生指标（公式）

| 指标     | 公式                                |
| -------- | ----------------------------------- |
| 活动率   | = 开单人数YTD ÷ 平均人力            |
| 件均保费 | = 承保新单规模保费YTD ÷ 承保件数YTD |
| 人均保费 | = 承保新单规模保费YTD ÷ 月末人力    |

#### 月度规模保费明细

除上述汇总区外，还在 G/H 列输出两家公司各自的月度规模保费序列（行13=1月，行14=2月，……，行24=12月）。超过报告月的月份填入 `#N/A`。

#### 列布局

源数据的列按"计划列 | 达成列"隔列排列：D列=1月达成, F列=2月达成, H列=3月达成, ...。第 m 月达成列号 = 2×m + 2。

### 4.3 商写数据汇总

**数据源**：`活动量/{公司}.xlsx` → Sheet `写字楼和商业综合体类`

**汇总公司**：北京中言、大连凯丹、福建钱隆、春夏秋冬、重庆宜新（5 家）

**目标位置**：输出文件 `商写类` Sheet，各公司按 C~G 列依次排列

#### 汇总指标

| 指标             | 计算方式                                |
| ---------------- | --------------------------------------- |
| 期初面积         | 直接取 1 月达成值                       |
| 新增签约面积 YTD | 1 月至报告月逐月累加                    |
| 平均租金         | 预留（固定填 0）                        |
| 本期退租面积 YTD | 1 月至报告月逐月累加                    |
| 月末面积         | 取报告月当月值，不累加                  |
| 渠道带客 YTD     | 1 月至报告月逐月累加                    |
| 渠道成交 YTD     | 1 月至报告月逐月累加                    |
| 渠道签约面积 YTD | 1 月至报告月逐月累加                    |
| 成交周期（渠道） | 预留（固定填 0）                        |
| 渠道转化率       | = 渠道成交YTD ÷ 渠道带客YTD（写入公式） |
| 自营带客 YTD     | 1 月至报告月逐月累加                    |
| 自营成交 YTD     | 1 月至报告月逐月累加                    |
| 自营签约面积 YTD | 1 月至报告月逐月累加                    |
| 成交周期（自营） | 预留（固定填 0）                        |
| 自营转化率       | = 自营成交YTD ÷ 自营带客YTD（写入公式） |
| 到期面积 YTD     | 1 月至报告月逐月累加                    |
| 续签面积 YTD     | 1 月至报告月逐月累加                    |
| 续签率           | 模板中预置公式，系统不写入              |

> 渠道转化率和自营转化率为 Excel 公式，当分母为 0 时 Excel 自动显示错误值，实际数据中出现全零指标时，后续 AI 分析会自动跳过。

### 4.4 酒店数据汇总

酒店业态的数据来源和布局最为复杂——两家公司的源数据布局不同，且需要从两份数据源（活动量、经营报表）分别提取。

**汇总公司**：伯豪瑞廷、重庆瑞尔（2 家）

**目标位置**：输出文件 `酒店类` Sheet

#### 两家公司的布局差异

| 差异项     | 伯豪瑞廷                            | 重庆瑞尔                            |
| ---------- | ----------------------------------- | ----------------------------------- |
| 达成列规律 | E列=1月, G=2月, I=3月, ...（2×m+3） | D列=1月, F=2月, H=3月, ...（2×m+2） |
| 投放数量   | 三行合计：官微 + 抖音 + OTA         | 单行                                |
| 受众数量   | 三行合计：官微 + 抖音 + OTA         | 单行                                |
| 成交数量   | 三行合计：官微 + 抖音 + OTA         | 单行                                |

#### 营销活动指标

| 指标         | 计算方式                                                   |
| ------------ | ---------------------------------------------------------- |
| 投放数量 YTD | 1 月至报告月，每个单元格提取数字后求和（伯豪瑞廷三行合计） |
| 受众数量 YTD | 同上                                                       |
| 成交数量 YTD | 同上                                                       |
| 转化率       | = 成交数量YTD ÷ 受众数量YTD（写入公式）                    |

#### 业务指标（从经营报表提取）

以下两项从各家酒店的经营报表 `指标统计` Sheet 中逐月提取（数据列号 = m + 3，D列=1月起始）：

| 指标         | 提取范围 | 说明                                  |
| ------------ | -------- | ------------------------------------- |
| 月均入住率   | 1～12 月 | 报告月前后全量输出，超出部分填 `#N/A` |
| OTA 网络评价 | 1～12 月 | 同上                                  |

业务指标的月度数据写入输出文件的 F:G 列（OTA 评分）和 J:K 列（入住率），行 2～13 分别对应 1 月～12 月。

### 4.5 经营报表汇总

**数据源**：`经营报表/{公司}.xlsx` → Sheet `指标统计`

**汇总公司**：按"填写页"C2:C10 列出的全部公司（通常 9 家）

经营报表汇总**不做任何计算、不修改数据格式**，执行的是纯值复制：

```
源数据范围: D4:O20（16行 × 12列，12列对应1～12月）
目标位置:   输出文件中同名 Sheet 的 G2:R18
```

源 Sheet `指标统计` 的布局：A列=指标类别，B列=指标名称，C列=年度目标，D~O列=1～12月数据。

## 六、AI 经营分析

AI 分析分为两个阶段，共计最多 12 段分析文案：

```
阶段一：板块级分析（按业态维度，共 3 段）
    ├── 商写业态分析 → 数据范围 "商写类" A1:G18
    ├── 保险业态分析 → 数据范围 "保险类" F1:H25
    └── 酒店业态分析 → 三个独立区域合并分析
         ├── 营销活动 B1:D5
         ├── OTA 评分 E1:G13
         └── 入住率   I1:K13

阶段二：公司级分析（按公司维度，共 9 段）
    逐公司读取各公司 Sheet 的 C1:R5 数据 → 各生成 1 段分析文案
```

### 6.1 API 调用配置

| 参数           | 板块分析        | 公司分析                      |
| -------------- | --------------- | ----------------------------- |
| 超时时间       | 30 秒           | 60 秒                         |
| 最大输出 Token | 1,000           | 1,500                         |
| Temperature    | 0.3             | 0.3                           |
| 质量评分       | 不评分          | 满分 10，≥8 分通过            |
| 重试机制       | HTTP 层指数退避 | HTTP 层 + 分析层最多 2 次重试 |

### 6.2 阶段一：板块级分析

**执行顺序**：商写 → 保险 → 酒店

板块分析不进行质量评分，仅校验输出长度 ≥ 50 字。

#### 保险板块分析

**数据过滤**：月度规模保费区域（行13-24）中，超过报告月的月份行会被自动过滤，确保 AI 不会看到 `#N/A` 对应的行。

**分析维度与要求**：

| 分析维度       | 关键指标                                   | 约束                         |
| -------------- | ------------------------------------------ | ---------------------------- |
| 人力与活动     | 月末人力、开单人数、活动率                 | 只陈述数值差异               |
| 承保新单及效率 | 承保新单规模保费、件数、件均保费、人均保费 | 指出哪家效率更高或结构更优   |
| 月度规模保费   | 1～报告月各月规模保费序列                  | 描述水平及环比变化，忽略空值 |

**输出格式**：首行摘要 ≤ 40字 → `人力与活动：...` → `承保新单及效率：...` → `月度规模保费：...`

#### 商写板块分析

**分析维度与要求**：

| 分析维度 | 关键指标                         | 约束                     |
| -------- | -------------------------------- | ------------------------ |
| 整体情况 | 新增签约面积、退租面积           | 不提及期初面积、月末面积 |
| 合作渠道 | 渠道带客、成交、签约面积、转化率 | 自动剔除全 0 指标        |
| 自有招商 | 自营带客、成交、签约面积、转化率 | 同上                     |
| 续租情况 | 到期面积、续签面积、续签率       | 同上                     |

**输出格式**：首行摘要 ≤ 40字 → `整体情况：...` → `合作渠道：...` → `自有招商：...` → `续租情况：...`

#### 酒店板块分析

**分析维度与要求**：

| 分析维度   | 关键指标                             | 约束             |
| ---------- | ------------------------------------ | ---------------- |
| 营销活动   | 投放数量、受众数量、成交数量、转化率 | 两家横向对比     |
| OTA 评分   | OTA 网络评价（月度序列）             | 忽略 `#N/A` 月份 |
| 月均入住率 | 月均入住率（月度序列）               | 忽略 `#N/A` 月份 |

**输出格式**：首行摘要 ≤ 40字 → `营销活动：...` → `OTA评分：...` → `月均入住率：...`

### 6.3 阶段二：公司经营指标分析

每家公司独立分析，数据范围为公司 Sheet 的 `C1:R5`（约前4～5个核心指标），分析结果写入 `C61` 单元格。

#### 分析上下文

发送给 AI 的信息包含：

- 公司名称
- 数据年份
- 当前月份
- 序时进度（如 33.33%）
- 数据单位：万元

#### 质量评分体系（满分 10 分，5 维 × 2 分）

| 维度     | 通过条件                                                     |
| -------- | ------------------------------------------------------------ |
| 摘要     | 首行不含"营业收入""EBITDA""扣非""经营活动""经营支出"等关键词 |
| 营业收入 | 任一行包含"营业收入"                                         |
| EBITDA   | 任一行包含"EBITDA"或"扣非利润"                               |
| 现金流   | 任一行包含"经营活动净现金流"或"现金流"                       |
| 经营支出 | 任一行包含"经营支出"                                         |

**通过阈值**：≥ 8 分。不通过时自动重试，最多 2 次，每次间隔 2 秒。重试后仍不达标则在文案末尾追加质量提示标记。

#### 输出规则

| 规则         | 说明                                                                    |
| ------------ | ----------------------------------------------------------------------- |
| 序时进度     | 以提供的值为准，不自行计算                                              |
| 达成率对比   | 达成率 > 序时进度 → "领先序时进度 X.X 个百分点"                         |
| 环比趋势     | 增→"环比上升"，减→"环比下降"                                            |
| 利润（负值） | 亏损收窄→"环比减亏"，亏损扩大→"环比增亏"                                |
| 支出/成本    | 增→"支出环比扩大"，减→"支出环比收窄"                                    |
| 成本管控判断 | 达成率明显低于序时进度→"成本管控有效"；接近/高于且波动小→"刚性成本较高" |
| 波动性       | (最大−最小) / 均值 > 30% → "波动大"，≤ 30% → "波动平稳"                 |
| 禁止项       | 主观情绪词、因果推断（"由于…导致…"）、改进建议                          |

#### 输出格式（每家公司）

- **首行摘要**：≤ 50 字，以 `[年份]年前[月份]个月，[公司名称]` 开头，抓住最突出的经营特征
- **后续行**：按数据表前四个指标的原始顺序，每行 ≤ 60 字
    - 收入/现金流/利润类：`指标名：累计 XX 万元，年度达成率 XX%，领先/落后序时进度 X.X 个百分点，环比趋势，波动性。`
    - 支出/成本类：`指标名：累计 XX 万元，年度达成率 XX%，环比趋势，波动性，[成本管控有效/刚性成本较高]。`
    - 利润目标为负值的指标使用专属表述：`实际累计亏损 XX 万元，较年度目标（亏损 XX 万元）减亏/超支 XX 万元`

#### 分批处理

每 3 家公司为一批，批次间延迟 1 秒，避免 API 限流。
