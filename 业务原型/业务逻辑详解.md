## 二、数据清洗规则

### 2.1 数字提取（ParseNumeric）

`汇总公共模块.bas` 的 `ParseNumeric` 函数处理优先级：

| 优先级 | 输入示例    | 处理逻辑                                                    | 结果    |
| ------ | ----------- | ----------------------------------------------------------- | ------- |
| 1      | 空单元格    | `IsEmpty(v)` → 0                                            | 0       |
| 2      | 纯数值      | `IsNumeric(v)` → `CDbl(v)` 直接取值                         | 12.5    |
| 3      | 文本含数字  | 逐字符扫描，收集连续的数字和第一个小数点，构建数字串后 CDbl  | 见下文  |
| 4      | 纯中文/其他 | 无数字→返回 0                                                | 0       |

**VBA ParseNumeric 实现**（`汇总公共模块.bas` Lines 93-121）：

```
逐字符扫描 CStr(v):
  若 ch 是数字(0-9)或 "." → 拼接到 numStr
  否则跳过
扫描完成后，若 numStr 非空且 IsNumeric → CDbl(numStr)，否则 0
```

**注意事项**：
- 文本中包含小数点的，仅保留第一个小数点
- 不处理百分号（VBA 原生不支持百分号解析）
- 不处理货币符号（¥、$ 等符号直接被跳过，只保留数字和 `.`）
- 不处理负号（`-` 被视为非数字字符跳过）
- `#N/A` 等错误值通过 `IsEmpty`/`IsNumeric` 检查后返回 0

### 2.2 数字提取增强版（ExtractNumberFromCell）

`酒店数据汇总.bas` 的 `ExtractNumberFromCell` 函数（Lines 175-216）新增了加法表达式支持：

| 优先级 | 输入示例      | 处理逻辑                                      | 结果  |
| ------ | ------------- | --------------------------------------------- | ----- |
| 1      | 空值          | `IsEmpty` → 0                                 | 0     |
| 2      | 错误值        | `IsError` → 0                                 | 0     |
| 3      | 纯数值        | `IsNumeric` → `CDbl`                          | 12.5  |
| 4      | 含 `+` 的文本 | 按 `+` 分割，每段调 `ExtractSingleNumber` 求和 | 1101  |
| 5      | 普通文本      | 调 `ExtractSingleNumber` 提取首个连续数字串    | 1736  |

**ExtractSingleNumber 实现**（Lines 219-250）：

```
逐字符扫描:
  数字(0-9) → 拼接到 numStr，foundDigit=True
  "." 且 foundDigit 且 !foundDot → 拼接到 numStr，foundDot=True
  已找到数字后遇到非数字非小数点的字符 → 终止扫描(Exit For)
```

示例：
- `"直播1736场"` → 跳过"直播" → 提取 `1736`
- `"3.5万"` → 提取 `3.5`
- `"1+1000"` → 分割为 `["1", "1000"]` → `1 + 1000 = 1001`
- `"85%"` → 扫描得到 `85`，不处理百分号

### 2.3 安全读取封装

`汇总公共模块.bas` 提供了三层读取函数：

| 函数                  | 行为                                                      |
| --------------------- | --------------------------------------------------------- |
| `SafeRead(ws,r,c)`    | `ParseNumeric(ws.Cells(r,c).Value)` — 提取数字，失败返0   |
| `SafeReadEx(ws,r,c)`  | 与 SafeRead 相同逻辑的别名                                |
| `ParseNumeric(v)`     | 从任意 Variant 中提取数字                                 |

### 2.4 日期解析

`汇总公共模块.bas` 的 `GetTargetMonth` 函数（Lines 22-43）确定报告月份的优先级：

1. 读取 `填写页!A2` 单元格
   - 若为数值 → `CLng(v)` 直接取整
   - 若为日期 → `Month(CDate(v))` 提取月份
2. 回退到文件夹名解析 `GetMonthFromFolderName`
3. 最终回退到当前系统月份 `Month(Date)`

`GetMonthFromFolderName` 解析逻辑（Lines 49-80）：
- 取 `ThisWorkbook.Path` 最末段文件夹名
- 定位 `年` 和 `月` 之间的数字字符 → 取整
- 如 `2026年4月` → 提取 `4`

## 三、数据汇总 — 四个业态

**VBA 源码**：[保险数据汇总.bas](保险数据汇总.bas)

**数据源**：`活动量/{公司}.xlsx` → Sheet `保险类`
**汇总公司**：盛唐融信、君康经纪（2 家）
**目标 Sheet**：`保险类`（与源 Sheet 同名，但位于汇总输出文件）

#### 源行号常量

| VBA 常量                | 行号 | 含义                 |
| ----------------------- | ---- | -------------------- |
| `ROW_BASE_HR`           | 4    | 期初人力             |
| `ROW_HR_IN`             | 5    | 当月入职             |
| `ROW_HR_OUT`            | 6    | 当月离职             |
| `ROW_OPEN_COUNT`        | 10   | 开单人数             |
| `ROW_NEW_PREMIUM`       | 12   | 承保新单规模保费     |
| `ROW_QJ_PREMIUM`        | 13   | 承保期交规模保费     |
| `ROW_AR_13`             | 14   | 应收13月续期保费     |
| `ROW_RC_13`             | 15   | 实收13月续期保费     |
| `ROW_AR_25`             | 16   | 应收25月续期保费     |
| `ROW_RC_25`             | 17   | 实收25月续期保费     |
| `ROW_POLICIES`          | 18   | 承保件数             |

#### 列号规律

达成列隔列排列（计划列 | 达成列），第 m 月达成列号 = `2×m + 2`：
- 1月达成 = D列（列号4）
- 2月达成 = F列（列号6）
- 3月达成 = H列（列号8）
- ...12月达成 = Z列（列号26）

#### 计算公式

**YTD 累加**（`SumAchievementCols` 函数）：
```
For m = 1 To numMonths
    col = 2 * m + 2
    total = total + ParseNumeric(wsSource.Cells(sourceRow, col).Value)
```

**人力派生指标**：
| 指标     | 公式                                                  |
| -------- | ----------------------------------------------------- |
| 当月净增 | `YTD入职 − YTD离职`                                   |
| 月末人力 | `期初人力 + YTD入职 − YTD离职`                         |
| 平均人力 | `Σ(各月月末人力) ÷ 月数`（逐月累加计算，非一次性除法）|

**效率派生指标**：
| 指标     | 公式                                             |
| -------- | ------------------------------------------------ |
| 活动率   | `开单人数YTD ÷ 平均人力`                          |
| 件均保费 | `承保新单规模保费YTD ÷ 承保件数YTD`               |
| 人均保费 | `承保新单规模保费YTD ÷ 月末人力`                  |

**续期保费**：13月/25月的应收和实收均为**报告月当月值**，不进行 YTD 累加。VBA 使用 `WriteCell`（保留原始值，含 Empty）写入，区别于 `WriteValue`（强制写入数值）。

#### 写入位置（目标 Sheet `保险类`）

**目标列**：盛唐融信→C列（`SHENGTANG_TARGET_COL=3`），君康经纪→D列（`JUNKANG_TARGET_COL=4`）

**汇总指标区**：

| 目标行 | 指标                 | C/D 列写入方式                               | 格式        |

| Row 2  | 期初人力             | `WriteValue(t,2,c, baseHR)`                  | #,##0.00    |

| Row 6  | 月末人力             | `WriteValue(t,6,c, endHR)`                   | #,##0.00    |
| Row 7  | 平均人力             | `WriteValue(t,7,c, avgHR)`                   | #,##0.00    |
| Row 8  | 开单人数 YTD         | `WriteValue(t,8,c, ytdOpenCount)`            | #,##0.00    |
| Row 9  | 活动率               | `=C8/C7` 公式                                | **0.00%**   |
| Row 10 | 承保新单规模保费 YTD | `WriteValue(t,10,c, sum_ach(12))`            | **#,##0.00**|
| Row 11 | 承保期交规模保费 YTD | `WriteValue(t,11,c, sum_ach(13))`            | **#,##0.00**|

| Row 16 | 承保件数 YTD         | `WriteValue(t,16,c, sum_ach(18))`            | #,##0.00    |
| Row 17 | 件均保费             | `=C10/C16` 公式                              | #,##0.00    |

> 格式设置代码：`C9:D9.NumberFormat = "0.00%"`、`C10:D11.NumberFormat = "#,##0.00"`、`C12:D18.NumberFormat = "#,##0.00"`

**月度规模保费区**（G/H列，G=盛唐融信，H=君康经纪）：

#### 3.3.2 辅助列机制（伯豪瑞廷/重庆瑞尔共用）

VBA 在源工作表的 AE 列（第31列）开始创建辅助区域，提取每个达成列单元格中的数字：

```
辅助列起始: AUX_START_COL = 31 (AE列)
辅助列布局: 连续排列 → AE=1月, AF=2月, AG=3月, ...

伯豪瑞廷版: srcCol = 2*m+3, auxCol = 31+(m-1)
重庆瑞尔版: srcCol = 2*m+2, auxCol = 31+(m-1)
```

每个辅助单元格通过 `ExtractNumberFromCell` 从源单元格提取数字（支持 `"直播1736场"→1736`、`"1+1000"→1001`）。

`SumAuxRow` 函数对辅助区域指定行的 1..numMonths 列求和得到 YTD 值。

#### 3.3.3 营销活动指标计算

**伯豪瑞廷**：
```
投放YTD = SumAuxRow(12) + SumAuxRow(13) + SumAuxRow(14)  // 官微+抖音+OTA
受众YTD = SumAuxRow(15) + SumAuxRow(16) + SumAuxRow(17)
成交YTD = SumAuxRow(18) + SumAuxRow(19) + SumAuxRow(20)
```

**重庆瑞尔**：
```
投放YTD = SumAuxRow(12)  // 单行
受众YTD = SumAuxRow(13)
成交YTD = SumAuxRow(14)
```

**转化率**：`=成交÷受众`，写入公式 `=col4/col3`

#### 3.3.4 业务指标部分

从经营报表 `指标统计` Sheet 提取月度序列，数据列号 = `m + 3`（D=4(1月), E=5(2月), ...）：

| 指标         | 源行号（常量）              | 数据范围                                  | 格式    |
| ------------ | --------------------------- | ----------------------------------------- | ------- |
| 月均入住率   | `REPORT_ROW_OCCUPANCY = 15` | 1~12月逐月（`for m=1..12: col=m+3`）      | **0%**  |
| OTA 网络评价 | `REPORT_ROW_OTA_RATING = 17`| 1~12月逐月，超出报告月填 `CVErr(xlErrNA)` | **0.00**|

#### 3.3.5 写入位置（目标 Sheet `酒店类`）

**营销活动区（A1:D5）**：

## 五、AI 经营分析

### 5.1 架构总览

**VBA 源码**：
- 板块分析：[业态分析整合版.bas](业态分析整合版.bas)
- 公司分析：[核心指标分析.bas](核心指标分析.bas)
- API 调用：[DeepSeekAPI.bas](DeepSeekAPI.bas)

AI 分析分为两个阶段，共计最多 12 段分析：

```
阶段一：板块级分析（按业态维度，3 段）
    ├── 商写业态分析 → 数据范围 A1:G18 → 1 段分析文案
    ├── 保险业态分析 → 数据范围 F1:H25 → 1 段分析文案
    └── 酒店业态分析 → 数据范围 B1:D5 + E1:G13 + I1:K13 → 1 段分析文案

阶段二：子公司经营指标分析（按公司维度，9 段）
    逐公司 → 9 家公司各自的 C1:R5 数据 → 各 1 段分析文案
```

### 5.2 API 配置参数

| 参数              | DeepSeekAPI.bas 默认值 | 核心指标分析.bas 覆盖值 | 说明                   |
| ----------------- | ---------------------- | ----------------------- | ---------------------- |
| API 地址          | `https://api.deepseek.com` | 不变                 | `/chat/completions`    |
| 模型              | `deepseek-chat`        | 不变                    |                        |
| Temperature       | 0.3                    | 0.3（不变）             |                        |
| Max Tokens        | **1000**               | **1500**                | 公司分析使用更大值     |
| Timeout (ms)      | **30000**              | **60000**               | 公司分析使用更大值     |
| HTTP 层重试次数   | 3                      | 1（由分析层控制）       | 避免嵌套重试           |
| HTTP 重试策略     | 指数退避 2^retry 秒   | 同                      |                        |

> **业态分析整合版** 使用 DeepSeekAPI.bas 默认参数（timeout=30s, max_tokens=1000）。

### 5.3 阶段一：板块级分析

**执行顺序**：商写 → 保险 → 酒店（VBA `运行所有业态分析` 的调用顺序）

#### 数据输入范围

| 业态 | Sheet    | 数据范围                  | 说明                                          |
| ---- | -------- | ------------------------- | --------------------------------------------- |
| 商写 | `商写类` | `A1:G18`                  | 包含项目名+5家公司全部16个指标                 |
| 保险 | `保险类` | `F1:H25`                  | 含月度规模保费序列，超出报告月的月份自动过滤  |
| 酒店 | `酒店类` | `B1:D5`+`E1:G13`+`I1:K13`| 三个独立区域分别对应营销活动、OTA评分、入住率  |

#### 保险业态数据过滤

保险业态分析的数据构建（`BuildInsuranceDataPrompt`）有特殊处理——规模保费区域（行13-24）中，月份标签超过当前报告月的行会被跳过滤（`monthNum > currentMonth → GoTo NextRow`）。这确保 AI 不会看到 `#N/A` 对应的月份行。

#### 板块分析特征

| 特征       | 具体行为                                               |
| ---------- | ------------------------------------------------------ |
| 系统提示词 | 从各业态 Sheet 的 L1（保险/商写）或 M1（酒店）单元格读取 |
| 质量检查   | **跳过**，仅校验内容长度 ≥ 50 字                        |
| 重试机制   | DeepSeekAPI.bas 的 HTTP 层重试（指数退避，最多3次）     |
| 结果写入   | 保险/商写→L14 单元格，酒店→M14 单元格                  |

### 5.4 阶段二：子公司经营指标分析

**VBA 源码**：[核心指标分析.bas](核心指标分析.bas)

#### 数据范围

- 数据源：各公司 Sheet 的 `C1:R5`（5行 × 18列）
- 结果写入：各公司 Sheet 的 `C61` 单元格 (`RESULT_CELL`)

#### 分析上下文

发送给 AI 的上下文信息：
```
公司名称：{companyName}
年份：{year}
当前月份：{month}月
序时进度：{progressPercent}%
数据单位：万元
```

年份从 `填写页!A4` 读取（支持日期、数值、文本格式），序时进度从 `填写页!A8` 读取（支持百分比和数值格式，若 ≤1 则自动 ×100）。

#### 批次处理

| 参数                | 常量             | 值      |
| ------------------- | ---------------- | ------- |
| 批次大小            | `BATCH_SIZE`     | 3       |
| 批次间延迟          | `BATCH_DELAY_MS` | 1000ms  |
| 应用暂停实现        | —                | `Application.Wait Now + TimeSerial(0,0,1)` |

每 3 家公司为一批，批次间停留 1 秒避免 API 限流。

#### 质量评分

5维评分（VBA `CheckAnalysisQuality` 函数），每项 2 分，满分 10 分：

| 维度       | 检测逻辑                                                                                              | 分值 |
| ---------- | ----------------------------------------------------------------------------------------------------- | ---- |
| 摘要       | 首行不含"营业收入""EBITDA""扣非""经营活动""经营支出"关键词                                            | 2    |
| 营业收入   | 任一行含"营业收入"                                                                                    | 2    |
| EBITDA     | 任一行含"EBITDA" 或 "扣非利润"                                                                        | 2    |
| 现金流     | 任一行含"经营活动净现金流" 或 "现金流"                                                                | 2    |
| 经营支出   | 任一行含"经营支出"                                                                                    | 2    |

```vba
' VBA 评分逻辑
If quality.HasSummary Then quality.Score = quality.Score + 2
If quality.HasRevenue Then quality.Score = quality.Score + 2
If quality.HasEbitda Then quality.Score = quality.Score + 2
If quality.HasCashFlow Then quality.Score = quality.Score + 2
If quality.HasExpense Then quality.Score = quality.Score + 2
```

#### 重试机制

| 配置             | 值                     |
| ---------------- | ---------------------- |
| 通过阈值         | **≥ 8 分**             |
| 最大重试次数     | **2 次**               |
| 重试间隔         | 2秒 (`RETRY_DELAY_MS`) |
| 重试验收条件     | Score ≥ 8 或 重试次数 > MAX_RETRIES |
| 未达标标记       | 追加 `[质量提示：本分析质量评分 X/10，部分指标描述可能不完整]` |

HTTP 层与分析层的重试关系：
- HTTP 层（DeepSeekAPI.bas）：最多3次，指数退避 2^retry 秒
- 分析层（核心指标分析.bas）：最多2次，固定2秒延迟
- 调用时 HTTP 层设为1次（`maxRetries=1`），避免嵌套重试

### 5.5 各业态分析维度（来自 VBA 提示词）

#### 保险板块分析

| 分析维度       | 关键指标                                   | 约束                                  |
| -------------- | ------------------------------------------ | ------------------------------------- |
| 人力与活动     | 月末人力、开单人数、活动率                 | 只陈述数值差异                        |
| 承保新单及效率 | 承保新单规模保费、件数、件均保费、人均保费 | 指出哪家效率更高或结构更优            |
| 月度规模保费   | 1~报告月各月规模保费                       | 描述保费水平及环比变化，忽略 #N/A     |

**输出格式**：首行摘要≤40字 → `人力与活动：...` → `承保新单及效率：...` → `月度规模保费：...`

#### 商写板块分析

| 分析维度 | 关键指标                         | 约束                                        |
| -------- | -------------------------------- | ------------------------------------------- |
| 整体情况 | 新增签约面积、退租面积           | 不提及期初面积、月末面积                    |
| 合作渠道 | 渠道带客、成交、签约面积、转化率 | 自动剔除全 0 指标                           |
| 自有招商 | 自营带客、成交、签约面积、转化率 | 同上                                        |
| 续租情况 | 到期面积、续签面积、续签率       | 同上                                        |

**输出格式**：首行摘要≤40字 → `整体情况：...` → `合作渠道：...` → `自有招商：...` → `续租情况：...`

#### 酒店板块分析

| 分析维度   | 关键指标                           | 约束               |
| ---------- | ---------------------------------- | ------------------ |
| 营销活动   | 投放数量、受众数量、成交数量、转化率 | 两家横向对比       |
| OTA 评分   | OTA 网络评价（月度序列）            | 忽略 #N/A 月份     |
| 月均入住率 | 月均入住率（月度序列）              | 忽略 #N/A 月份     |

**输出格式**：首行摘要≤40字 → `营销活动：...` → `OTA评分：...` → `月均入住率：...`

#### 子公司经营指标分析

**分析规则**（来自 `经营分析提示词.md`/`PROMPT_FINANCIAL`）：

| 规则         | 说明                                                                                  |
| ------------ | ------------------------------------------------------------------------------------- |
| 序时进度     | 以填写的值为准，不自行计算                                                            |
| 达成率对比   | 收入/净现金流/利润类：达成率 > 序时进度 → "领先 X.X 个百分点"                          |
| 环比趋势     | 收入/现金流：增→"环比上升"，减→"环比下降"                                             |
| 利润(负值)   | 亏损收窄→"环比减亏"，亏损扩大→"环比增亏"                                              |
| 支出/成本    | 增→"支出环比扩大"，减→"支出环比收窄/缩减"                                             |
| 支出判断     | 达成率明显低于序时进度→"成本管控有效"；接近/高于序时进度且波动小→"刚性成本较高"       |
| 波动性       | (max-min)/均值 > 30%→"波动大"，≤30%→"波动平稳"                                        |
| 禁止项       | 主观情绪词、因果推断（"由于…导致…"）、改进建议                                       |

**输出格式**（每家公司）：
- **首行摘要**：≤50字，以 `[年份]年前[月份]个月，[公司名称]` 开头
- **后续行**：按数据表前4个指标原始顺序，每行≤60字
  - 收入/现金流/利润类：`指标名：累计 XX 万元，年度达成率 XX%，领先/落后序时进度 X.X 个百分点，环比趋势，波动性。`
  - 支出/成本类：`指标名：累计 XX 万元，年度达成率 XX%，环比趋势，波动性，[成本管控有效/刚性成本较高]。`

### 5.6 API 响应解析

DeepSeekAPI.bas 的 `ParseAPIResponseSimple` 从 JSON 响应中提取 content 字段：
1. 定位 `"content":"` 位置（`CONTENT_PREFIX_LEN=11`）
2. 从该位置扫描，追踪转义 `\` 状态，直到找到未转义的 `"`
3. 提取中间文本，反转义：`\"→"`, `\n→CRLF`, `\t→Tab`, `\/→/`, `\\→\`

提取后通过 `CleanAnalysisResult` 去除空行（逐行 Trim，非空才保留）。

- `WrapText = True`
- `HorizontalAlignment = xlLeft`
- `VerticalAlignment = xlTop`

## 六、API 调用公共模块

**VBA 源码**：[DeepSeekAPI.bas](DeepSeekAPI.bas)

### 6.1 HTTP 客户端

创建 `MSXML2.ServerXMLHTTP.6.0` 对象，失败回退 `MSXML2.XMLHTTP`。

### 6.2 JSON 构建

请求体格式：
```json
{
  "model": "deepseek-chat",
  "messages": [
    {"role": "system", "content": "<system prompt>"},
    {"role": "user", "content": "<user prompt>"}
  ],
  "temperature": 0.3,
  "max_tokens": 1000
}
```

JSON 字符串在 VBA 中手动拼接，通过 `EscapeJSONString` 转义特殊字符（`\`, `"`, 换行, Tab 等）。

### 6.3 重试逻辑

`CallDeepSeekAPI` 的 HTTP 层重试：
- 最多 3 次（`MAX_RETRIES=3`）
- 指数退避：`waitSec = 2^retryCount`
- 实现方式：`Application.Wait (Now + TimeValue("0:00:" & CStr(waitSec)))`
- 不可重试状态码：401, 402, 403, 422

### 6.4 API Key 管理

从 `~/.dskey`（即 `%USERPROFILE%\.dskey`）读取，格式为：

```
[EXCEL]=sk-xxxxxxxx
```

VBA 逐行扫描文件，查找 `EXCEL=` 前缀的行并提取后面的值。

### 6.5 数据提示词构建

`BuildDataPrompt(dataArray)` 将 VBA 二维数组转换为管道分隔的表格文本：
```
数据表格如下：
| 指标1 | 值1 | 值2 | ... |
| 指标2 | 值3 | 值4 | ... |
```
