# ExcelMiner 业务逻辑详解

> 本文档基于 VBA 原型脚本（`业务原型/*.bas`），完整描述数据从 Excel 源文件到最终分析文案的全链路业务处理逻辑。不含任何技术实现细节。

## 二、数据清洗规则

### 2.1 数字提取

从 Excel 单元格中提取数值的规则（优先级从高到低）：

| 优先级 | 输入示例           | 处理逻辑                                               | 结果    |
| ------ | ------------------ | ------------------------------------------------------ | ------- |
| 1      | `85%` / `85.5%`    | 识别百分号，除以 100                                   | 0.85    |
| 2      | `1+1000`           | 识别加号，各部分分别提取数字后求和                     | 1001    |
| 3      | `¥1,234.56`        | 去除货币符号、中文文字、千分位逗号，提取首个连续数字串 | 1234.56 |
| 3      | `直播1736场`       | 同上，跳过前导中文                                     | 1736    |
| 3      | `3.5万`            | 同上（"万"不参与计算，仅提取 3.5）                     | 3.5     |
| 4      | 纯中文/空白/错误值 | 无法提取数字                                           | 0       |

**补充说明**：

- 文本中包含小数点的，仅保留第一个小数点，后续非数字字符结束提取
- 空白单元格和 `#N/A` 等错误值均视为 0
- 酒店业态的数据源中，部分单元格含文字描述（如"直播1736场"），需要从中提取数字

### 2.2 日期解析

报告月份的确定优先级：

1. 读取 `填写页!A2` 单元格（支持纯数字 `4` 或日期格式）
2. 从汇总文件所在文件夹名解析（如 `2026年4月` → 找到"年"和"月"之间的数字 → `4`）
3. 回退到当前系统月份

YTD（年初至今）累计范围：从 1 月至报告月的所有月份。

## 三、数据汇总 — 四个业态

### 3.1 保险数据汇总

**数据源**：`活动量/{公司}.xlsx` → Sheet `保险类`
**汇总公司**：盛唐融信、君康经纪（2 家）
**指标总数**：16 个（含 3 个派生指标 + 1 组月度序列）

**源数据表格结构**：

- 每行一个指标，1月~12月的"达成"列隔列排列（D=1月达成, F=2月达成, H=3月达成...）
- 列号规律：第 m 月的达成列 = `2×m + 2`（如 1 月→D 列，2 月→F 列）

#### 指标明细

| 序号 | 指标                 | 源行号 | 计算方式                                          |
| ---- | -------------------- | ------ | ------------------------------------------------- |
| 1    | 期初人力             | Row 4  | = 1 月达成值（固定取 1 月，不累加）               |
| 2    | YTD 入职             | Row 5  | = Σ（1 月入职达成…报告月入职达成），逐月累加      |
| 3    | YTD 离职             | Row 6  | = Σ（1 月离职达成…报告月离职达成）                |
| 4    | 当月净增             | —      | = YTD 入职 − YTD 离职（派生）                     |
| 5    | 月末人力             | —      | = 期初人力 + YTD 入职 − YTD 离职（派生）          |
| 6    | 平均人力             | —      | = Σ（各月月末人力）÷ 月数（派生）                 |
| 7    | 开单人数 YTD         | Row 10 | = Σ（1 月…报告月）                                |
| 8    | 承保新单规模保费 YTD | Row 12 | = Σ（1 月…报告月）                                |
| 9    | 承保期交规模保费 YTD | Row 13 | = Σ（1 月…报告月）                                |
| 10   | 续期 13 月应收       | Row 14 | = 报告月达成值（当月值，不累加）                  |
| 11   | 续期 13 月实收       | Row 15 | = 报告月达成值                                    |
| 12   | 续期 25 月应收       | Row 16 | = 报告月达成值                                    |
| 13   | 续期 25 月实收       | Row 17 | = 报告月达成值                                    |
| 14   | 承保件数 YTD         | Row 18 | = Σ（1 月…报告月）                                |
| 15   | 活动率               | —      | = 开单人数 YTD ÷ 平均人力（公式派生）             |
| 16   | 件均保费             | —      | = 承保新单规模保费 YTD ÷ 承保件数 YTD（公式派生） |
| 17   | 人均保费             | —      | = 承保新单规模保费 YTD ÷ 月末人力（公式派生）     |

**月度规模保费序列**（额外）：

- 取 Row 12（承保新单规模保费）的 1~12 月各月达成值，逐月输出
- 超出报告月的月份填 `#N/A`

**续期保费说明**：

- 续期 13 月 / 25 月的应收和实收均为报告月当月值，不进行 YTD 累加
- 反映的是当月到期的续期保费回收情况

### 3.3 酒店数据汇总

**数据源**：两份独立数据源

- 营销活动数据：`活动量/{公司}.xlsx` → Sheet `酒店类`
- 业务指标数据：`经营报表/{公司}.xlsx` → Sheet `指标统计`

**汇总公司**：伯豪瑞廷、重庆瑞尔（2 家）
**指标总数**：6 个

#### 3.3.1 营销活动部分

两家公司的源数据布局不同，需要分别处理：

| 差异项     | 伯豪瑞廷                                                | 重庆瑞尔        |
| ---------- | ------------------------------------------------------- | --------------- |
| 达成列起始 | E 列（`2×m+3`）                                         | D 列（`2×m+2`） |
| 投放数量   | 三行合计：官微（Row 12）+ 抖音（Row 13）+ OTA（Row 14） | 单行：Row 12    |
| 受众数量   | 三行合计：官微（Row 15）+ 抖音（Row 16）+ OTA（Row 17） | 单行：Row 13    |
| 成交数量   | 三行合计：官微（Row 18）+ 抖音（Row 19）+ OTA（Row 20） | 单行：Row 14    |
| 转化率     | = 成交 ÷ 受众                                           | = 成交 ÷ 受众   |

**数据解读**：

- 投放数量：酒店在各渠道（微信公众号、抖音、OTA 平台）的营销内容投放量
- 受众数量：营销内容触达的受众规模
- 成交数量：通过营销转化带来的实际成交量
- 转化率 = 成交 / 受众，反映营销效率

**写入位置（营销活动区 `A1:D5`）**：

| 目标行 | 指标     | 伯豪瑞廷 | 重庆瑞尔 | 说明                                       |
| ------ | -------- | :------: | :------: | ------------------------------------------ |
| Row 2  | 投放数量 |    C2    |    D2    | 伯豪瑞廷=三行合计(YTD)，重庆瑞尔=单行(YTD) |
| Row 3  | 受众数量 |    C3    |    D3    | 同上                                       |
| Row 4  | 成交数量 |    C4    |    D4    | 同上                                       |
| Row 5  | 转化率   | `=C4/C3` | `=D4/D3` | 公式，格式 0.0000%                         |

> C2:D4 区域设置千分位格式 `#,##0`。

### 3.4 经营报表汇总

**数据源**：`经营报表/{公司}.xlsx` → Sheet `指标统计`
**汇总公司**：全部 9 家子公司
**指标数**：8 个核心经营指标

**读取区域**：`D4:O20`（16 行 × 12 列，12 列对应 1~12 月）

**8 个核心指标**：

| 序号 | 指标             | 说明                     |
| ---- | ---------------- | ------------------------ |
| 1    | 营业收入         | 企业主营业务产生的收入   |
| 2    | 营业成本         | 与营业收入直接相关的成本 |
| 3    | 管理费用         | 行政管理部门发生的费用   |
| 4    | 销售费用         | 销售活动产生的费用       |
| 5    | 净利润           | 收入扣除成本费用后的净额 |
| 6    | EBITDA           | 息税折旧摊销前利润       |
| 7    | 经营活动净现金流 | 经营活动现金流入 − 流出  |
| 8    | 经营支出         | 经营活动产生的总支出     |

**业务逻辑**：

- 经营报表汇总**不做任何计算**，直接按 `D4:O20` 区域全量复制各公司的原始月度数据
- 每家公司按 `填写页!C2:C10` 中列出的名称，依次打开对应的 `经营报表/{公司名}.xlsx` 进行数据提取
- 汇总后的数据供 AI 逐公司进行经营指标分析

#### 写入位置（各公司独立 Sheet）

> 汇总输出文件中，每家公司拥有一个以公司名命名的独立 Sheet（如 `盛唐融信`、`伯豪瑞廷` 等），Sheet 的格式和布局已事前设计好。

| 目标区域 | 源区域                   | 说明                        |
| -------- | ------------------------ | --------------------------- |
| `G2:R18` | 源文件 `指标统计!D4:O20` | 16 行 × 12 列，直接以值粘贴 |

> 即：源数据的 D4:O20 → 目标 Sheet 的 G2:R18。数据不做任何加工，仅变换位置。

## 四、汇总数据存放

### 4.1 数据流转路径

```
Excel 源文件（活动量/ & 经营报表/）
    │
    ▼
四个业态的汇总计算（保险/商写/酒店/经营报表）
    │
    ├──→ 填入汇总输出文件中各 Sheet 的指定位置（数值 + 公式）
    │       （Sheet 的格式、布局、标题均已事前设计好）
    │
    └──→ 传递至 AI 分析环节，作为分析数据源
```

### 4.2 xlsx 输出文件 Sheet 结构

汇总输出文件（如 `[2026年4月] 经营数据.xlsx`）是一个事前设计好格式和布局的模板文件，宏将汇总数据填入指定位置：

| Sheet 名                                  | 汇总数据填入位置                                                                  | 说明             |
| ----------------------------------------- | --------------------------------------------------------------------------------- | ---------------- |
| `填写页`                                  | A2（报告月份）、A4（年份）、A8（序时进度）、C2:C10（公司列表）、C20（分析提示词） | 业务参数配置     |
| `保险类`                                  | C2:D18（盛唐融信 & 君康经纪 17 项指标）、G13:H24（月度规模保费序列 12 个月）      | 见 §3.1 写入位置 |
| `商写类`                                  | C2:G18（5 家公司 16 项指标）                                                      | 见 §3.2 写入位置 |
| `酒店类`                                  | C2:D5（营销活动 4 项）、F2:G13（OTA 评分 12 个月）、J2:K13（入住率 12 个月）      | 见 §3.3 写入位置 |
| `盛唐融信` ~ `重庆瑞尔`（9 个公司 Sheet） | G2:R18 ← 源 `指标统计!D4:O20`                                                     | 见 §3.4 写入位置 |

## 五、AI 经营分析

### 5.1 两阶段分析架构

AI 分析分为两个阶段执行：

```
阶段一：板块级分析（按业态维度，3 段）
    ├── 保险板块分析 → 合并盛唐融信 + 君康经纪数据 → 1 段分析文案
    ├── 商写板块分析 → 合并 5 家公司数据 → 1 段分析文案
    └── 酒店板块分析 → 合并伯豪瑞廷 + 重庆瑞尔数据 → 1 段分析文案

阶段二：子公司经营指标分析（按公司维度，9 段）
    逐公司 → 9 家公司各自 → 各 1 段分析文案
```

**总计最多 12 段分析**（3 板块 + 9 公司）。

### 5.2 各业态分析维度

#### 保险板块分析

| 分析维度       | 关键指标                                   | 分析要点                                 |
| -------------- | ------------------------------------------ | ---------------------------------------- |
| 人力与活动     | 月末人力、开单人数、活动率                 | 两家公司横向对比，仅陈述数值差异         |
| 承保新单及效率 | 承保新单规模保费、件数、件均保费、人均保费 | 指出哪家效率更高或结构更优，不分析原因   |
| 月度规模保费   | 1~报告月各月规模保费                       | 描述保费水平及环比变化，忽略 `#N/A` 月份 |

**输出格式**：

- 首行：摘要（≤40字），概括两家公司最突出的业务差异
- 后续：按 `人力与活动：` `承保新单及效率：` `月度规模保费：` 三个小标题分段输出（每段≤50字）
- 禁止：分析原因、提建议、主观情绪词

#### 商写板块分析

| 分析维度 | 关键指标                         | 分析要点                              |
| -------- | -------------------------------- | ------------------------------------- |
| 整体情况 | 新增签约面积、退租面积           | 横向对比各公司，不得提及期初/月末面积 |
| 合作渠道 | 渠道带客、成交、签约面积、转化率 | 横向对比，自动剔除全 0 指标           |
| 自有招商 | 自营带客、成交、签约面积、转化率 | 同上                                  |
| 续租情况 | 到期面积、续签面积、续签率       | 同上                                  |

**输出格式**：

- 首行：摘要（≤40字），概括最突出的经营特征，不得提期初/月末面积
- 后续：按 `整体情况：` `合作渠道：` `自有招商：` `续租情况：` 四个小标题分段输出（每段≤50字）
- 自动剔除全 0 指标：若某指标在所有公司中均为 0，整段不提
- 对比用语："均高于""明显低于""最大/最小""仅 XX 公司有"等中性表述

#### 酒店板块分析

| 分析维度   | 关键指标                             | 分析要点                             |
| ---------- | ------------------------------------ | ------------------------------------ |
| 营销活动   | 投放数量、受众数量、成交数量、转化率 | 两家酒店横向对比，仅陈述数值差异     |
| OTA 评分   | OTA 网络评价（月度序列）             | 描述每家评分变化，忽略 `#N/A` 月份   |
| 月均入住率 | 月均入住率（月度序列）               | 描述每家入住率变化，忽略 `#N/A` 月份 |

**输出格式**：

- 首行：摘要（≤40字）
- 后续：按 `营销活动：` `OTA评分：` `月均入住率：` 三个小标题分段输出

#### 子公司经营指标分析

每家公司独立分析，基于 8 个核心经营指标：

**分析规则**：

| 规则         | 说明                                                                                                             |
| ------------ | ---------------------------------------------------------------------------------------------------------------- |
| 序时进度     | 以填写的值为准，不自行计算                                                                                       |
| 达成率对比   | 收入/净现金流/利润类：达成率 > 序时进度 → "领先 X.X 个百分点"；达成率 < 序时进度 → "落后 X.X 个百分点"           |
| 环比趋势     | 收入/现金流：增→"环比上升"，减→"环比下降"                                                                        |
| 利润（负值） | 亏损收窄→"环比减亏"，亏损扩大→"环比增亏"；实际亏损 < 目标亏损→"减亏 XX 万元"；实际亏损 > 目标亏损→"超支 XX 万元" |
| 支出/成本    | 增→"支出环比扩大"，减→"支出环比收窄/缩减"                                                                        |
| 支出判断     | 达成率明显低于序时进度→"成本管控有效"；接近/高于序时进度且波动小→"刚性成本较高"。不直接对比支出达成率与序时进度  |
| 波动性       | (最大值 − 最小值) ÷ 均值，>30%→"波动大"，≤30%→"波动平稳"                                                         |
| 禁止项       | 主观情绪词、因果推断（"由于…导致…"）、改进建议                                                                   |

**输出格式**（每家公司）：

- **首行摘要**：≤50字，以"[年份]年前[月份]个月，[公司名]"开头，抓住最突出特征
- **后续行**：按数据表前四个指标的原始顺序，每行一个指标（≤60字）
    - 收入/现金流/利润类：`指标名：累计 XX 万元，年度达成率 XX%，领先/落后序时进度 X.X 个百分点，环比趋势，波动性。`
    - 支出/成本类：`指标名：累计 XX 万元，年度达成率 XX%，环比趋势，波动性，[成本管控有效/刚性成本较高]。`

### 5.3 分析质量标准

#### 5 维度质量评分（满分 10 分）

经营指标分析对每段输出进行 5 维度质量检查：

| 维度     | 检查内容                                                       | 分值 |
| -------- | -------------------------------------------------------------- | ---- |
| 摘要     | 首行是否为非指标行（不含"营业收入""EBITDA"等关键词）的概括描述 | 2    |
| 营业收入 | 是否包含"营业收入"相关内容                                     | 2    |
| EBITDA   | 是否包含"EBITDA"或"扣非利润"相关内容                           | 2    |
| 现金流   | 是否包含"经营活动净现金流"相关内容                             | 2    |
| 经营支出 | 是否包含"经营支出"相关内容                                     | 2    |

板块分析按各自的分析维度（小标题）进行类似的维度覆盖检查：

- 保险：检测是否含"人力""承保/新单""保费/规模"
- 商写：检测是否含"整体/情况""合作/渠道""招商/自有""续租"
- 酒店：检测是否含"营销/活动""OTA/评价/评分""入住率/入住"

#### 质量保障机制

- **最低质量线**：评分 ≥ 8 分才算达标
- **自动重分析**：未达标时自动重新分析，最多重试 2 次
- **降级标记**：重试用尽仍未达标，保留最后结果并追加 `[质量提示：本分析质量评分 X/10，部分指标描述可能不完整]`
- **批次控制**：每 3 家公司为一批，批次间短暂延迟，避免请求过载

### 5.4 板块分析数据合并

阶段一的板块分析会将同业态所有公司的汇总数据合并到一起提交 AI：

- 保险板块：合并盛唐融信 + 君康经纪的全部 16 个指标数据
- 商写板块：合并 5 家公司全部的 14 个指标数据（含各公司之间的对比）
- 酒店板块：合并伯豪瑞廷 + 重庆瑞尔的营销活动数据 + OTA 评分序列 + 入住率序列

### 5.5 分析结果输出

分析结果按三大板块分别写入汇总文件的三个业态 Sheet，子公司经营指标分析结果写入各公司对应 Sheet，同时整合到 `AI分析结果` Sheet 供统一查阅。

| 信息项    | 内容示例                              |
| --------- | ------------------------------------- |
| 公司/板块 | `保险板块` / `盛唐融信`               |
| 分析类别  | `板块分析` / `公司经营分析`           |
| 业态      | `保险` / `商写` / `酒店` / `经营指标` |
| 质量评分  | `8/10`                                |
| 分析内容  | 完整的 AI 分析文本                    |

## 六、报表导出

### 6.1 导出内容

汇总输出文件（`[YYYY年M月] 经营数据.xlsx`）包含以下内容：

| Sheet        | 内容来源                                    |
| ------------ | ------------------------------------------- |
| `填写页`     | 报告月份、年份、项目名称等元数据            |
| `保险类`     | 保险汇总 — 盛唐融信 & 君康经纪 16 项指标    |
| `商写类`     | 商写汇总 — 5 家公司 14 项指标               |
| `酒店类`     | 酒店汇总 — 2 家公司营销 + 入住率 + OTA 评分 |
| `AI分析结果` | AI 分析 — 板块分析 + 公司经营分析全文       |

### 6.2 数据展平规则

汇总数据中的以下字段因其包含嵌套的月度序列，在展平输出时作为独立数据块处理（不作为普通键值对输出）：

- `月度规模保费`（保险，12 个月数组）
- `月均入住率按月`（酒店，12 个月数组）
- `OTA 网络评价按月`（酒店，12 个月数组）
- `indicators`（经营报表，全量指标表）

其余字段展平为 `父项.子项` 格式输出。

## 附录：各业态分析提示词要点

### 保险分析提示词

- **角色**：严谨的保险中介经营数据分析师
- **输出要求**：摘要 ≤40字 + 三段小标题（人力与活动 / 承保新单及效率 / 月度规模保费）
- **约束**：仅陈述差异、不分析原因、不提建议、不使用主观词
- **数据缺失**：`#N/A` 或空白跳过不提及

### 商写分析提示词

- **角色**：严谨的商写租赁经营分析师
- **输出要求**：摘要 ≤40字 + 四段小标题（整体情况 / 合作渠道 / 自有招商 / 续租情况）
- **特殊规则**：不提及期初面积和月末面积；自动剔除全 0 指标
- **对比用语**：使用"均高于""明显低于""最大/最小""仅 XX 公司有"等中性表述

### 酒店分析提示词

- **角色**：严谨的酒店经营数据分析师
- **输出要求**：摘要 ≤40字 + 三段小标题（营销活动 / OTA评分 / 月均入住率）
- **约束**：仅陈述数值差异、不分析原因、不提建议

### 财务分析师提示词

- **角色**：严谨的高级财务分析师
- **输出要求**：摘要 ≤50字（以"[年份]年前[月份]个月，[公司名]"开头）+ 逐指标行分析
- **核心规则**：序时进度引用不自行计算、达成率对比、环比趋势、波动性判断
- **利润类特殊处理**：目标为负时按绝对值表述，用"减亏/增亏"
- **支出类特殊处理**：不直接对比达成率与序时进度，仅做内部判断（"成本管控有效"/"刚性成本较高"）
